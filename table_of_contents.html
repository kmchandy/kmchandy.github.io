Detection Algorithms

Algorithms by which an agent detects that the system is in a stable state. Examples of such algorithms are termination detection and deadlock detection. These algorithms are based on the concept of consistent cuts.
  1. Termination Detection An application of consistent cuts of timeline diagrams: detecting that a computation has terminated.
  2. Deadlock Detection I Algorithm to detect deadlock in systems in which an idle agent is made active when it receives a message from all agents for which it is waiting. This algorithm is based on consistent cuts. The algorithm can be used to detect deadlock in database systems.
  3. Diffusing Computations Algorithms in which an agent starts a computation across a network of agents; detects that the computation has terminated; and collects information from participating agents.
  4. Applications of Diffusing Computations Learning structures of networks in which an agent is embedded. Termination detection. Communication deadlock detection.

Resilient Algorithms

Algorithms that are resilient to errors. Algorithms that use varying amounts of resources. Algorithms that use incentives for agents to participate.
  1. The Paxos Algorithm Achieving consensus among agents in the presence of error.
  2. Self Stabilization Algorithms Self-stabilizing algorithms automatically correct themselves in response to transient faults.
  3. Byzantine Algorithms Algorithms that reach consensus among a set of faulty agents who may corrupt messages.
  4. Distributed ledgers: Block chain.
  5. Distributed algorithms in the cloud. Map-Reduce.
