Timelines, Global Snapshots, Logical Clocks

This section introduces the concept of timelines and timeline diagrams. The section defines cuts of timelines. The concept of a valid cut of a timeline is used to define global snapshots and timestamps assigned by logical clocks. This section describes algorithms for logical clocks and snapshots for simple action systems and for distributed systems.
  1. Timeline Diagrams The concept of a timeline defined for an action system consisting of a set of actions where an action reads and writes variables.
  2. Message-Passing Channels Timeline diagrams for systems in which shared variables are message-passing channels.
  3. Global Snapshots with Channels Consistent cuts, consistent timestmaps, and the global snapshot algorithm for the case where shared variables are message-passing channels.

Detection Algorithms

Algorithms by which an agent detects that the system is in a stable state. Examples of such algorithms are termination detection and deadlock detection. These algorithms are based on the concept of consistent cuts of timelines.
  1. Termination Detection An application of consistent cuts of timeline diagrams: detecting that a computation has terminated.
  2. Deadlock Detection I Algorithm to detect deadlock in systems in which an idle agent is made active when it receives a message from all agents for which it is waiting. This algorithm is based on consistent cuts. The algorithm can be used to detect deadlock in database systems.

What an Agent Knows, Diffusing Computations

  1. Concept of knowledge: What an agent knows. What does an agent know about the states of other agents? About the states of channels? How does this knowledge change as the agent sends and receives messages?
  2. Diffusing Computations Algorithms in which an agent starts a computation across a network of agents; detects that the computation has terminated; and collects information from participating agents.
  3. Applications of Diffusing Computations Learning structures of networks in which an agent is embedded. Termination detection. Communication deadlock detection.

Resilient Algorithms

Algorithms that are resilient to errors. Algorithms that use varying amounts of resources. Algorithms that use incentives for agents to participate.
  1. Self Stabilization Algorithms Self-stabilizing algorithms automatically correct themselves in response to transient faults.
  2. Reaching Consensus in a Finite Number of Steps is Impossible This module gives the intuition for why consensus among distributed agents cannot be reached when message delays are arbitrarily long.
  3. The Paxos Algorithm The Paxos algorithm is a best-effort algorithm for reaching consensus among distributed agents in the presence of errors and unbounded message delays. The algorithm does not guarantee that consensus will be reached; however, if consensus is reached then it has desirable properties.
  4. Byzantine Algorithms: Written Messages Algorithms that reach consensus among a set of faulty agents who may corrupt messages. This algorithm assumes that messages are encrypted.
  5. Byzantine Algorithms: Oral Messages Algorithms that reach consensus among a set of faulty agents who may corrupt messages. This algorithm deals with messages that may be forged.
