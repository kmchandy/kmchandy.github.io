Invariants

The state of a do-od loop is given by the values of program variables. A computation of the loop is a sequence of states. The same do-od loop may have multiple computations because the selection of the next guarded command to execute is nondeterministic.

An initial state of the computation satisfies the initial condition of the loop. Each execution of a guarded command in which the guard is true causes a state transition. If execution of the loop terminates then the terminal state is one in which all the guards are false. If execution of the loop does not terminate then the corresponding computation is infinite.

An invariant is a predicate that holds in every state of every computation.

We can prove that predicate \(P\) is an invariant by showing that
  1. \(P\) holds initially, and
  2. If \(P\) holds in a state before a state transition then \(P\) holds in the state after the state transition, i.e., for every command \(s\): \( \{P\} \; s \{P\} \)

are given in Examples of Correctness Proofs.

Summary

We will use invariants to prove the correctness of distributed algorithms. We prove that a predicate is an invariant of a distributed algorithm in the same way as for a sequential algorithm.
