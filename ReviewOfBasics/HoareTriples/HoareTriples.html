Hoare Triples

This is a very short, limited review of properties of Hoare triples that you probably studied in earlier courses.
A Hoare triple has the form \(\{P\} c \{Q\}\) where \(P\) and \(Q\) are predicates on states of a system and \(c\) is a command. The meaning of the triple is that if \(P\) holds when the command \(c\) is initiated then execution of the command terminates and \(Q\) holds upon termination.
\(P\) is called the precondition and \(Q\) the postcondition of the triple

(Note: We assume total rather than partial correctness.)

Examples

Examples of Hoare triples for the assignment statement \(x := x+1\) are:

\( \{x = 0\} \; x := x+1 \; \{x = 1\} \)

\( \{x = 0\} \; x := x+1 \; \{x > 0\} \)

And a set of Hoare triples:

\( \forall k: \quad \{x = k\} \; x := x+1 \; \{x = k+1\} \)

Variables not in the Hoare Triple

Variables that are not mentioned in a Hoare triple are assumed, implicitly, to remain unchanged by the command. For example, if a program has variables \(x\) and \(y\), then the triple:

\( \{x = 0\} \; x := x+1 \; \{x = 1\} \)

is implicitly equivalent to:

\( \forall k: \quad \{(x = 0) \wedge (y=k) \} \; x := x+1 \; \{(x = 1) \wedge (y = k)\} \)

Examples

Consider the following Hoare triples.
  1. \( \{x < 0\} \; c \; \{(x = 1) \vee (x = 2)\} \)
  2. \( \{x > 0\} \; c \; \{(x = 0) \vee (x = -1)\} \)
  3. \( \{x > 1\} \; c \; \{x < 0\} \)
What transitions are possible from a state in which \(x = -1\) by a command \(c\) that satisfies all the above three Hoare triples?

Since the state satisfies the first precondition, \(x < 0\), the state after the transition must satisfy the first postcondition, \((x = 1) \vee (x = 2)\). The other two Hoare triples don't address the precondition \(x = -1\), and so the postconditions of these Hoare triples are irrelevant for this question. Therefore, the transitions from \(x = -1\) are to states \(x = 1\) or \(x = 2\).

Example

What transitions are possible from a state in which \(x = 2\) that satisfy all of the above Hoare triples?

The first Hoare triple doesn't address the precondition \(x = 2\).

The precondition \(x > 0\) of the second triple holds when \(x = 2\) and therefore its postcondition, \((x = 0) \vee (x = -1)\), must hold in the next state.

The precondition \(x > 1\) of the third triple holds when \(x = 2\) and therefore its postcondition, \(x < 0\), must hold in the next state.

Both postconditions, \((x = 0) \vee (x = -1)\) and \(x < 0\) must hold in the next state. So the only transition from \(x = 2\) is to state \(x = -1\).

Example

What transitions are possible from a state in which \(x = 0\)?

None of the three preconditions hold when \(x = 0\), and therefore the next state can be any state.

Example: Unimplementable Commands

What transitions satisfy all the following Hoare triples?

\( \{x = 0\} \; c \; \{x > 0\}, \quad \{x = 0\} \; c \; \{x < 0\} \)

This command cannot be initiated in a state in which \(x = 0\) because both postconditions \(x > 0\) and \(x < 0\) must hold when the command terminates.

In this course we will ensure that the commands specified in our algorithms can be executed in every state.

Weakest Precondition for Assignment

What is the largest set of states in which execution of the assignment 
x = x + 1
results in a state in which x > 1?

This question is equivalent to: What is the weakest preconditon \(P\) that satisfies the Hoare triple \(\{P\} x := x + 1 \{x > 1\}\)?

The answer is obtained by substituting the right-hand side (i.e. \(x+1\)) of the assignment for the left-hand side (i.e., \(x\)) of the assignment in the postcondition (i.e., \(\{x > 1\}\)).

With this substitution we get:

\( x + 1 > 1 \)

Which simplifies to \(x > 0\). Therefore we get the Hoare triple:

\( \{x > 0\} x := x + 1 \{x > 1\} \)

Any precondition that is stronger than the weakest precondition satisfies the Hoare triple.

In terms of sets of states, any subset of the largest set (i.e., extension of \(x > 0\)) also satisfies the Hoare triple. For example:

\( [(x = 5) \Rightarrow (x > 0)] \)

and therefore \(\{x = 5\} x := x + 1 \{x > 1\}\).

Example

What is the weakest precondition \(P\) that satisfies the following Hoare triple?

\( \{P\} x := 2*x + 3*y \{5*x + y > 25\} \)

Substitute \(2*x+3*y\) for \(x\) in \(5*x + y > 25\) to get:

\( 5*(2*x + 3*y) + y > 25 \), or

\( 10*x + 16*y > 25 \)

So, we have the Hoare triple: \( \{10*x +16*y > 25\} x := 2*x + 3*y \{5*x + y > 25\} \)

Example

Is the following Hoare triple correct?

\( \{(x = 1) \wedge (y =1)\} x := 2*x + 3*y \{5*x + y > 25\} \)

Yes, because \((x = 1) \wedge (y =1)\) is stronger than \(10*x +16*y > 25\), i.e.,

\( [(x = 1) \wedge (y =1) \quad \Rightarrow \quad (10*x +16*y >25)] \)

You can also verify the state transition directly. If the above assignment is executed in a state in which \((x = 1) \wedge (y =1)\) then in the next state \((x = 5) \wedge (y =1)\) which satisfies the postcondition.

Summary

We describe algorithms in terms of state transitions and predicates that hold at points in a computation. We use Hoare triples to relate commands defined in terms of states to commands implemented in a programming language.
