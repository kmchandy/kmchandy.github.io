Nondeterministic Iteration

This module is a review of nondeterministic iteration implemented as do-od loops. A do-od loop consists of a set of guarded commands. A guarded command consists of a guard and a command, and we use the syntax: 
guard -> command
The execution of the loop is as follows: Execute any guarded command of the loop for which the guard evaluates to True. Execution of the loop terminates when all guards evaluate to False.

Loops in most programming languages are deterministic: the next statement to be executed is uniquely determined by the current state of the program. A do-od loop is nondeterministic: Multiple guards may be true and the next statement to be executed can be any guarded command with a true guard.

Example: Sorting

Let \(X\) be an array of \(N\) numbers. The following loop sorts the array in ascending order. (We assume that indices start at 0.) 
for all i where 0 < i < N:
   X[i-1] > X[i]   ->    X[i-1], X[i] = X[i], X[i-1]
The loop has \(N-1\) guarded commands. Execution of the loop proceeds by executing any of the \(N-1\) guarded commands for which the guard is true. So, execution of the loop proceeds by flipping any adjacent pair of elements that is out of order. Execution of the loop terminates when all guards are False, at which point the array is in order.
