Predicates on States

The state of a program is given by the values of its variables. We reason about the correctness of programs by reasoning about sets of program states. For example, we will prove properties of a program such as: all states in all computations that start from any state in a set init of states are in a set safe.

We use operations, such as union and intersection, on sets of states. For example, for a program with variables x, y, z the set safe may be defined as the intersection of the set of states in which x > y and the set in which y > z.

In this course, a predicate is a function from states of a program to the Boolean constants \(\{True, False\}\). For example, for a program with variables x, y, the predicate x > y is true in the state (x, y) = (1, 0) and is false in the state (x, y) = (1, 1).

Set of elements associated with a predicate

A predicate \(P\) on states of a program defines a set of states for which \(P\) holds. For example, for a program with variables (x, y), the state (x, y) = (1, 0) is a member of the set of states for which the predicate x > y whereas (x, y) = (0, 1) is not.

Disjunction (or, \(\vee\)) and conjunction (and, \(\wedge\)) of predicates correspond to the union \(\cup\) and intersection \(\cap\), respectively, of the corresponding sets of states. True is a predicate whose extension is the universal set, and False is a predicate whose extension is the empty set.
