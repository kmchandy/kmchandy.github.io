Partially Synchronous Algorithms

In this section we develop a method that allows us to design some distributed algorithms so that we can reason about the correctness of the algorithms assuming that messages are delivered instantaneously and agents operate synchronously. Reasoning about algorithms in which agents operate in lock step is often easier than designing asynchronous algorithms. The central idea is assigning a sync-time to each event so that events in which a message is sent and received have the same sync-times. So, in terms of sync-times, messages are instantaneous.

Synch-Times

Associated with each agent P is a local variable P.t which is called P's synchronous time or synch time. We associate a sync-time e.t with each event e. If event e occurs at agent P then e.t is the value of P.t at the point at which e occurs. With each message M we associate a field M.t called the message's sync-time, where M.t is the sync-time of the event in which M is sent.

As a trajectory grows, an agent's sync-time may remain unchanged or increase, but never decreases. Therefore, in a trajectory, earlier events at an agent have sync-times that are less than or equal to sync-times of later events at the same agent. It is possible, however, for earlier events in a trajectory at one agent to have sync-times that are greater than sync-times of later events at a different agent.

The action of receiving a message occurs in two steps:

  1. Message Delivery: A message is delivered to the receiver which inspects the message but takes no action other than to modify the receiver's sync-time.
  2. Message Processing: The receiver executes the event due to the arrival of the message. The receiver may change its state and send messages in this event.
We design algorithms in which sync-times have the following property.

Key property of sync-times: The sync-time of an event in which a message M is sent is the the sync-time of the event in which M is processed.

Rules that ensure the sync-times property

Let d be the event in which a message M is sent, and let e be the event in which M is received. Let P and Q be the agents that send and receive M, respectively.
  1. When P sends M, it sets M.t to P.t, which is also the sync-time d.t of this event. So d.t = M.t
  2. If Q receives M where M.t < Q.t, it does not process the message. So Q's state does not change when M is received and there is no event associated with processing M.
  3. When Q receives M where M.t \(\geq\) Q.t, it may or may not process the message, depending on the specific algorithm. If Q does process the message, Q first sets Q.t to M.t and next executes event e that processes M. So e.t = M.t.
The proof that the key property of sync times, d.t = M.t = e.t, is straightforward.

Theorem: Synchronous Versions of Asynchronous Computations

Let X be a trajectory of a system in which agents have sync-times that satisfy the above rules. Let Y be a permutation of X where the timelines of each agent are the same in X and Y, and events appear in increasing order of sync-times in Y. Then Y is also a trajectory of the system.

Proof

An event e depends on an event e' only if e'.t \(\leq\) e.t. A sequence of events in which each event occurs after all events on which it depends is also a trajectory.
