Termination Detection

A distributed computation has terminated when all agents are idle and all channels are empty. A termination detection algorithm is executed by the operating system to determine whether a client computation has terminated. Detection algorithms are based on global snapshots.

Problem Definition

Review of Agents

We described agents earlier. We give a summary of that description next.

An agent is either idle or active. An idle agent remains idle until it receives a message at which point it becomes active. An idle agent does not send messages. An active agent may send messages. An active agent may become idle at any point.

A computation terminates in states in which all agents are idle and all channels are empty. Termination is stable -- if a computation has terminated then it remains terminated.

Detection of Stable Properties: Specification

The problem is to develop an operating system algorithm that detects whether a stable property holds in a client's computation. In this example the stable property to be detected is termination. See Global Snapshots for a description of OS algorithms.

The detection algorithm has a boolean variable \(B\) which is initially \(False\) and becomes \(True\) if and when the algorithm detects that the specified stable property holds. The specification of an algorithm to detect a stable property is as follows.

  1. If the stable property holds at a point \(t\) then \(B\) becomes \(True\) at or after \(t\).
  2. If \(B\) is \(True\) at any point then the stable property holds at that point.

General Method for Detecting Stable Properties

The general method for detecting stable properties is by monitoring a system by taking repeated snapshots. \(B\) becomes \(True\) if and when a snapshot shows that the stable predicate holds. See Global Snapshots and Logical Clocks .

Next we describe an algorithm for the specific case of termination detection. This algorithm is a straightforward application of. the Global Snapshot Theorem.

A Termination Detection Algorithm

The algorithm uses an agent, that we call the detector which is part of the operating system. Let \(C_{s}\) and \(C_{r}\) be the numbers of messages sent and received (respectively) on channel \(C\),.

Each time an agent transits from active to idle it sends a message to the detector. This message contains \(C_{s}\) for each of that agent's outgoing channels \(C\), and \(C_{r}\) for each of the agent's incoming channels \(C\).

The detector keeps only the latest message that it receives from each agent. Let \(C_{s}^{*}, C_{r}^{*}\) be the last value of \(C_{s}, C_{r}\) (respectively) that the detector has received.

The Detection Condition

The detector detects that computation has terminated if it has received at least one message from each agent and for all channels \(C\): \(C_{s}^{*} = C_{r}^{*}\).

Proof of Correctness
The proof of part 1 -- after the computation terminates the detection condition holds -- is straightforward. Now we prove part 2: If the detection condition holds then the computation has terminated.

Assume that the detection condition holds. Then agents sent messages to the detector with the values \(C_{s}^{*} = C_{r}^{*}\). Call the local snapshot in which an agent sends these messages the detection snapshot.

From the global snapshot theorem, local snapshots form a global snapshot if, for all channels \(C\): \(C_{s} \geq C_{r}\). This condition holds when \(C_{s} = C_{r}\). Therefore, the detection snapshots form a global snapshot. In this global snapshot all agents are idle and all channels are empty.
