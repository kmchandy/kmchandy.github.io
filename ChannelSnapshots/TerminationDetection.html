Termination Detection

A distributed computation has terminated when all agents are idle and all channels are empty. A termination detection algorithm is executed by the operating system to determine whether a client computation has terminated. Detection algorithms are based on global snapshots.

Problem Definition

Review of Agents and Termination of Computation

We described agents earlier. We give a summary of that description next.

An agent is either idle or active. An idle agent remains idle until it receives a message at which point it becomes active. An idle agent does not send messages. An active agent may send messages. An active agent may become idle at any point.

A computation terminates in states in which all agents are idle and all channels are empty. A channel \(C\) is empty exactly when \(\; C_{s} = C_{r}\), where \(C_{s}, C_{r}\) are the numbers of messages sent and received (respectively) on \(C\). So,

A computation is in a terminated state exactly when all agents are idle and for all channels \(C\), \(\; C_{s} = C_{r}\).

Termination is stable -- if a computation has terminated then it remains terminated. Termination detection is an example of detection of stable properties.

Detection of Stable Properties: Specification

The problem is to develop an operating system algorithm that detects whether a stable property holds in a client's computation. In this example the stable property to be detected is termination. See Global Snapshots for a description of OS algorithms.

The detection algorithm has a boolean variable \(detected\) which is initially \(False\) and becomes \(True\) if and when the algorithm detects that the specified stable property holds. The specification of an algorithm to detect a stable property is as follows.

  1. If the stable property holds at a point \(t\) then \(detected\) becomes \(True\) at or after \(t\).
  2. If \(detected\) is \(True\) at any point then the stable property holds at that point.

How Should You Approach the Problem?

Use the General Method for Detecting Stable Properties
The obvious way to detect stable properties is to use the general method: monitor a system by taking repeated snapshots. \(detected\) becomes \(True\) if and when a snapshot shows that the stable predicate holds. See Global Snapshots and Logical Clocks for ways to take repeated snapshots.
An Alternate Method for Detecting Termination
Next we describe another algorithm which is straightforward application of. the Theorem about Past and Future. From the theorem, a slice is is closed if: For all channels \(C\), \(\; C_{s} \geq C_{r}\), where \(C_{s}, C_{r}\) are the numbers of messages sent and received (respectively) on the channel in the past events.

So, all slices in which all agents are idle and for all channels \(C\), \(\; C_{s} = C_{r}\), are closed. This insight suggests the following algorithm.

A Termination Detection Algorithm

The algorithm uses an agent, that we call the detector which is part of the operating system. Let \(C_{s}\) and \(C_{r}\) be the numbers of messages sent and received (respectively) on channel \(C\),.

Each time an agent transits from active to idle it sends a message to the detector. This message contains \(C_{s}\) for each of that agent's outgoing channels \(C\), and \(C_{r}\) for each of the agent's incoming channels \(C\).

The detector keeps only the latest message that it receives from each agent. Let \(C_{s}^{*}, C_{r}^{*}\) be the last value of \(C_{s}, C_{r}\) (respectively) that the detector has received.

The Detection Condition

The detector detects that computation has terminated if it has received at least one message from each agent and for all channels \(C\): \(C_{s}^{*} = C_{r}^{*}\).

Proof of Correctness
Proof of part 1 -- If the computation terminates, then after the computation terminates the detection condition holds.

The set of the last messages sent by agents before they remains idle forever has the property \(C_{s}^{*} = C_{r}^{*}\) for all channels \(C\).

Proof of part 2 -- If the detection condition holds then computation has terminated.

If that the detection condition holds then agents sent messages to the detector with the values \(C_{s}^{*} = C_{r}^{*}\). The slice of points at which these messages are sent is closed. So, the state at the slice -- computation has terminated -- is a state in the computation.
