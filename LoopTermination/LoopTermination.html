Review of Loop Termination

This module reviews how loop variants also called variant functions are used to prove that execution of a loop terminates. This concept is related to directed graph models of state-transition systems.

Variant Functions

We use the term variant function, rather than loop variant, because we will use the concept to prove properties of systems that are not loops in programs. A variant function is a function from states of a system to a well-founded set, i.e. a set on which we can carry out induction. An example of a variant function is one from states to a set of integers which is bounded below, where the base case of the induction is the lower bound. The steps by which induction is carried out will be self-evident for the problems we study.

Let \(f\) be a variant function. Then \(f = k\), for a constant \(k\) is a predicate. The extension of this predicate is the set of states \(s\) where \(f(s) = k\). We use a variant function \(f\) in the following way.

We prove that execution of any command with a True guard reduces \(f\). So, the execution of every iteration reduces \(f\). Since \(f\) is bounded below, it cannot decrease for ever. Therefore, at some point in the computation all guards are False. And, at that that point execution of the loop terminates.

Let \(I\) be an invariant of the loop. Let the \(j\)-th guarded command have guard \(g_{j}\) and command \(c_{j}\). Let \(f\) be a variant function of the loop. We will prove that the execution of any command with a True guard reduces the variant function. We do so by proving the following Hoare triple for all \(j\):

\(\forall K: \quad \{(f = K) \wedge I \wedge g_{j}\} \; c_{j} \; \{f < k\} \)

Therefore each iteration reduces \(f\). If we view each iteration as a step in a state-transition system, then from the Hoare triples we conclude that every step reduces \(f\).

\( \forall K: \quad (f=K) \wedge I \wedge (\exists j: g_{j}) \quad \rightarrow \quad f < K \)

Because \(f\) is a variant function, its value cannot decrease over an unbounded number of steps.

How to prove termination of a loop

To summarize, we prove termination of a do-od loop in the following way.

(1) Identify a variant function and invariant that satisfies the above equation
(2) Show that the execution of any command with a True guard reduces the value of the variant function.

Variant Functions and State-Transition Graphs

A termination state of the loop is one without outgoing edges; it is a trapping state. The state-transition graph has no cycles because every edge of the graph is from a state with a higher value of \(f\) to a state with a lower value. The graph shows a partial order where states with higher \(f\)-values are higher in the partial order. The termination states are the bottom of the order.

Every edge from every path in the graph goes from higher to lower \(f\) values. Since \(f\) cannnot decrease forever and \(f\) maps states to a well-founded set, a path reaches a terminal state in a finite number of steps.

The invariant \(I\) holds at the beginning and end of each iteration. Command \(c_{j}\) is enabled exactly when \(g_{j}\) is True. So, a precondition for \(c_{j}\) to be executed is \(I \wedge g_{j}\). Therefore, we prove that for all \(K\):

\( \{I \wedge g_{j} \wedge (f = K)\} \; c_{j} \; \{f < k\} \)

Example: Sorting

A variant function is the total number of out-of-order pairs: \( |(j < k) \wedge (X[j] > X[k])| \)

It is easy to verify that each change in \(X\) reduces the number of out-of-order pairs.
