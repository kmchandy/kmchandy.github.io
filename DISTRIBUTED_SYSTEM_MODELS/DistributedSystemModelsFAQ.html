States, Transitions and Computations FAQ

Many distributed systems have continuous states and transitions. The model only deals with discrete states and transitions. Can algorithms for systems with continuous states be described using the model?

This website describes only a few algorithms for systems with continuous state. The models used for these systems are described later.

Systems with continuous state are critical to almost every aspect of life, and we will point to papers in which distributed systems are described.

The model only considers states in which all agents are idle. Isn't that too restrictive.

Yes, the model only considers states in which all agents are idle. And the model only considers state transitions in which one message is received. We will see in the following webpages that the dataflow of system trajectories are preserved in the simple model.

Many properties of system behavior can be determined by analyzing the flow of data between agents. Yes, the model is restrictive but adequate for our purposes because we prove properties of distributed computations by analyzing dataflow.

Shouldn't computations start from initial states?

Yes, we define a computation as a sequence of states that starts in any state. We explicitly consider computations that start in an initial state of the system when we need to do so. For example, we define invariant properties of a system as properties that hold in all states of computations that start in an initial state.

Treating a computation as any path in the state transition graph simplifies some analyses, and isn't restrictive.

The model is unfair. It allows one agent to keep executing state transitions while all the other agents remain idle forever. Isn't such a model ineffective?

Yes, the model is unfair. The model allows an infinite computation in which an agent communicates with itself forever, while other agents never change state. Yes, this is too weak to describe most algorithms. There are, however, a few algorithms that can be analyzed by determining properties of all computations including unfair ones. We describe these algorithms next.

Later, we look at computations in which certain messages are delivered eventually, and we describe a temporal logic that deals with eventualities.
