Events and Dataflow: FAQ

Does the definition of events depend on the receive function being deterministic?

Does the definition of events assume implicitly that the receive function is deterministic? Can the receive function have different outputs -- next state and output messages -- with the same inputs?

The receive function may be nondeterministic. An event represents a single execution of receive. The event specifies the outputs and inputs for that one execution.

Two executions of receive, with the same inputs, that produce different outputs are represented by two different events each of which shows the different outputs for the same inputs.

Is a dataflow defined for a computation or for a system?

The example of dataflow on the webpage with operations are \(+, -, / \) show the flow of data between operations in a system. The example of dataflow for a computation shows the flow of data between events in a computation. A system may have many computations and so may have many dataflow graphs. So, is dataflow defined for a system or a computation?

In these notes, we define dataflow for a computation and not for a system. A distributed system may have many computations and therefore many dataflow graphs.

The properties of dataflow given on the webpage hold whether dataflow is defined for a system or a computation.

Is the initial state represented by a single initial event or by a separate event for each agent?

The initial state is represented by a single event; however, it is easier to see the flow of data in a computation if this single event is represented by a set of initial vertices, one for each agent.

Is dataflow, as specified here, the same as assuming that execution of a receive is atomic?

Yes, each event is atomic. The execution of a receive by one agent does not change the execution of a receive by another agent. Dataflow, as described here, assumes that receives are atomic operations.
