Basics

This page describes a simple model for distributed algorithms. The model, though simple, is adequate for designing many algorithms. More models are introduced when needed.

Models of Distributed Systems

Complex Systems and Simple Models

Distributed systems are complex. The information system that supports a bank supports actions taking place in ATM machines, branch offices, and fraud analysis centers. An earthquake monitoring system has thousands of sensors and agents connected to sensors and in the cloud.

A model of a distributed system is an abstraction that ignores some features of systems while retaining others. Different models are used to design algorithms in different settings. A model of a system in which messages may be corrupted is different from one in which messages cannot be corrupted. We begin with a very simple model, identify its limitations, and show why it is useful.

Notations

Languages and software libraries for implementing distributed systems have many features because distributed systems are complex. The algorithms in this course deal with only a few key problems in distributed computing. We use a message queuing notation to describe algorithms: a message is appended to the tail of a queue and removed from its head. We give examples of implementations of the algorithms using libraries such as Advanced Message Queuing Protocol (AMQP) , and a Python implementation of AMQP.

A Simple Model: Agents and Channels

A distributed system consists of a set of agents and a set of channels. A channel is directed from one agent to one agent. A channel from an agent P to an agent Q is an output of P and an input of Q. An agent can send messages on its output channels and receive messages on its input channels. A system is represented by a directed graph in which vertices represent agents and edges represent channels.

A distributed system is initiated with sets of agents and channels that remain unchanged. Agents and channels are not created or deleted.

Message Communication: Channels

The state of a channel is the sequence of messages in the channel -- these are the messages that have been sent on the channel and that have not been received. An agent sends a message by appending the message to the tail of the sequence of messages in the channel. An agent can receive the message at the head of a nonempty sequence of messages in any of the agent's input channels. When an agent receives a message it is deleted from the sequence.

Associated with each channel is its initial state -- the sequence of messages in the channel when the distributed system starts executing.

Agents

Associated with each agent is (1) a program that initializes the variables of the agent and (2) a function 
receive(message, sender)
called a callback function in message queuing libraries. The parameters message and sender of the function are the message that is received, and the sender of the message. If an agent has a nonempty input channel then the receive function of the agent is called where message is the message at the head of the channel and sender is the agent that sent the message; the message is removed from the channel.

An agent sends a mesage by executing 

send(message, receiver)
The first parameter of send is the message that is sent, and the second paramenter is the agent to which the message is sent. Execution of this statement places the message in the output channel directed from the sender to the receiver. A message can be sent on a channel independent of the state of the channel. An agent is never blocked from sending a message.

Example of an Agent

This is an example of an agent that receives messages from agents pos and neg, and sends messages to agent results. The state of the agent is given by the values of its local variable sum and its program counter. 
// Initial state
sum = 0

def receive(message, sender):
   if sender == pos:
       sum = sum + message
   else:
       sum = sum - message
   send(sum, results)

See an example of an implementation in Python. The implementation is a simple simulation of a distributed system. Implementations with multiple processes using AMQP are given later.

Restrictions on Agents

An agent can only access that agent's variables, send messages only on that agent's output channels, and receive messages only on that agent's input channels. A receive function must not call itself, and every execution of receive must terminate.
