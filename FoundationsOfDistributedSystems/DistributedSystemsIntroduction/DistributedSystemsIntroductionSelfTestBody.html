Distributed Systems: Introduction. Self Test

Question 1

A system has two agents \(A\) and \(B\), and channels \(x\), \(y\) from agent \(A\) to agent \(B\), and channel \(z\) from \(B\) to \(A\).

The system has four indivisible tokens with ids 0, 1, 2, 3. Tokens are indivisible and cannot be created or destroyed. A token's id remains unchanged.

Initially channel \(x\) contains tokens with ids 0, 1, 2 with id 0 ahead of id 1 ahead of id 2. Initially channel \(z\) contains the token with id 3. Initially channel \(y\) is empty, and initially agents \(A\) and \(B\) hold no tokens.

Agent \(A\)'s actions are as follows. When it receives a token with id 0, 1, or 2 it sends the token immediately on channel \(x\). When it receives a token with id 3, agent \(A\) immediately sends the token on channel \(y\).

When agent \(B\) receives a token it immediately sends the token out on channel \(z\).

True or False? The very first token token that \(B\) receives has id 0.

Question 4

Consider the same system as in question 1.

Let \(P\) be the predicate: channel \(y\) contains the token with id 3 OR channel \(z\) contains the token with id 3. Let \(init\) be the predicate that defines the initial state.

True or False? \( [init \; \Rightarrow \; always(P)] \)

Answer to Question 1

False.

The token with id 3 can move from channel \(z\) to channel \(y\) and then be received by agent \(B\) before tokens in channel \(x\) are delivered to \(B\). So, the token with id 3 may be received first.

Answer to Question 2

False

The state of a channel is a queue. The order in which messages are received along a channel is the same as the order in which messages are sent along the channel. So, the tokens in channel \(z\) have to follow the cyclic order
0 -> 1 -> 2 -> 0 -> 1 -> 2 -> 0 -> 1 -> 2 -> ....
interspersed with token with id 3.

Answer to Question 3

True

Same reasoning as for question 2.

Answer to Question 4

True

\(P\) holds in all states in all trajectories that start in an initial state, and therefore \([init \; \Rightarrow \; always(P)]\).

Answer to Question 5

False.

Recall that \(next(P)\) holds for a state \(s\) exactly when in all paths from \(s\) the next state satisfies \(P\). Equivalently, \(next(P)\) holds for a state \(s\) exactly when in all transitions from \(s\) are to states in which \(P\) holds. The key point is "all transitions" as opposed to "there exists transitions."

In terms of Computation Tree Logic, \(next(P) = AX(P)\) where \(A\) stands for all and \(X\) stands for \(neXt\).

So, the question is this: Are all transitions from the state shown in fig. 3 to the state shown in fig. 4?

The answer to this question is No because there exists a transtion from the state shown in fig. 3 to the state shown in fig. 5. This transition occurs when the token with id 0 on channel \(x\) is received by agent \(B\) and put on channel \(z\).

So not all transitions from states in which \(P\) holds are to states in which \(Q\) holds.
