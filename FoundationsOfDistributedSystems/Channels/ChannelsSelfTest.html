Question 1

The figure below shows a timeline diagram with two agents \(A\) and \(B\) and 4 messages in transit along a channel \(c\) from \(A\) to \(B\). The points at which messages are sent are events (vertices) on \(A\)'s timeline. Likewise, the points at which messages are received are events on \(B\)'s timeline.

Each message is represented by a labeled directed edge where the label is a pair: the channel on which the message is sent, and the message itself. For example, message \(m_{0}\) is represented by an edge directed from the event that sends the message to the event that receives the message, and the edge is labeled with the message and the channel. We don't show the label for the channel \(c\) in the diagram so as not to crowd the diagram.

The distance along the horizontal line, for the timeline of each agent, indicates the real time elapsed since the trajectory was initiated. The diagram shows a point on the timeline diagrams of agents \(A\) and \(B\) after \(T\) seconds have elapsed.

Distributed System Timelines: Self Test

Question 1

True

The state at an instant in time is depicted by a vertical cut. All vertical cuts in this timeline cut 0, 1, or 2 message lines. 2.

Question 2

True The diagram shows a consistent cut where there are three messages in the channel in the global state corresponding to the cut.

Question 3

False.

Message \(m_{1}\) sent after the cut, i.e., after point \(P\), is received before the cut, i.e., after point \(Q\). So there is an edge from an event after the cut to an event before the cut.

You can see a message line crossing the cut boundary from outside to inside. The cut boundary is shown as a blue line in the figure.

Another way of answering the question is that the cut is inconsistent because there is an edge from the future (i.e. after the cut) to the past (i.e. before the cut).

Question 4

True

Every edge is from a lower-timestamp event to a higher-timestamp event.

Question 5

False

There is an edge from a higher-timestamp (4) event to a lower-timestamp event (3).

Question 6

True

We can get \(traj'\) from \(traj\) by permuting the order of independent events, keeping the order of dependent events unchanged.

Another way of answering this question is to observe that the cut with after events \(1, 2, 3, 4\) and before events \(5, 6, 7, 8\) is a valid cut.

Question 7

False

Event 6 (receive message \(m_{1}\)) occurs before event 2 (send message \(m_{1}\)), and so this sequence is not a valid trajectory.

Event 6 is dependent on event 2, and there order is permuted in \(traj'\).
